# Implementation Backlog

## Phase 1: Foundation

- Create HR Platform Settings. Done in Slice 01.
- Create Service Catalog and Approval Matrix. Done in Slice 01.
- Create reusable lifecycle/status fields and naming standards.
- Add core setup records after the DocTypes exist. Initial roles and default services are created during install in Slice 01.

## Phase 2: Workforce Core

- Employee extension model for Saudi-specific data.
- Contract, dependent, emergency contact, document, and expiry tracking.
- Numbering and onboarding defaults.

## Phase 3: Employee Services

- Unified Employee Service Request. First approval and fulfilment slice done in Slice 01.
- Service-specific child tables for visas, letters, insurance, travel, assets, and payments.
- Workflow and fulfilment tasks.

## Phase 4: Time to Pay

- Time Device and Device Sync Batch.
- Attendance Exception and Shift Change Request.
- Overtime Request and payroll adjustments.
- Bank Transfer Batch.

## Phase 5: Exit and Accruals

- Exit Case and Clearance Checklist.
- End Of Service Calculation and Final Settlement Run.
- Accrual Policy, Accrual Schedule, and Accrual Journal Proposal.

## Phase 6: Analytics and Compliance

- Expiry dashboards.
- Payroll, attendance, leave, service request, and settlement dashboards.
- Scheduled notifications and weekly summaries.
